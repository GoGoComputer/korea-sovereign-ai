# A2. 완전 수동 설치 — 이 레포가 자동으로 하는 일을 손으로

**🌐 Language**: **한국어 (현재)** · [English](en/A2-manual-install.md)

> 이 문서는 `install.sh` / `web-install.sh` 가 자동으로 처리하는 모든 단계를 **한 줄씩 직접** 실행하는 방법을 설명합니다.
> "안에서 무슨 일이 일어나는지 알고 깔고 싶다", "스크립트 권한 없는 회사 노트북에서 깐다", "다른 모델로 응용하고 싶다" 같은 분께 추천.
>
> 처음이면 → [A1-개념사전.md](A1-개념사전.md) 먼저 읽고 오세요.

## 📋 전체 흐름 한눈에

```
0. 환경 점검
1. Xcode CLT (git, 컴파일러)         ← 1번만
2. Ollama 설치                        ← 1번만
3. Python + 필수 라이브러리           ← 1번만
4. 모델 저장 폴더 생성                ← 1번만
5. EXAONE — Ollama로 받기
6. EXAONE — MLX로 변환
7. A.X 4.0 Light — MLX로 변환
8. Solar Pro 22B — GGUF 받아서 Ollama 등록
9. 단축 명령(PATH) 등록
10. (옵션) 자동 정리 cron
```

각 단계는 독립적이라 **원하는 것만** 해도 됩니다 (예: A.X 만 쓸 거면 5~6 건너뛰기).

## 0. 환경 점검 (1분)

터미널에서:

```bash
# 칩 확인 — "arm64" 나와야 함
uname -m

# macOS 버전 — 13 이상 권장
sw_vers -productVersion

# 메모리 (GB) — 16 이상 권장, Solar는 24 권장
sysctl -n hw.memsize | awk '{print int($1/1024/1024/1024) " GB"}'

# 디스크 여유 — 25GB 이상 비어 있어야 전체 설치 가능
df -h ~ | awk 'NR==2 {print "여유: " $4}'
```

`arm64` 가 아니거나 메모리 16GB 미만이면 일부 모델(특히 Solar)은 무리. EXAONE / A.X 만 시도하세요.

## 1. Xcode Command Line Tools (한 번만, 5~10분)

```bash
xcode-select --install
```

창이 뜨면 **[설치]** 클릭. 끝나면 확인:

```bash
git --version          # git version 2.x...
clang --version
```

> ❓ 뭐예요? Apple이 제공하는 개발자 기본 도구 모음. `git`, 컴파일러 등이 들어있음.

## 2. Ollama 설치 (한 번만, 2분)

### 방법 A — 공식 사이트 (가장 쉬움)
1. <https://ollama.com/download> 에서 **macOS** 클릭
2. 다운로드된 `Ollama.zip` → 압축 해제 → `Ollama.app` 을 **응용 프로그램** 으로 드래그
3. `Ollama.app` 실행 (메뉴바에 라마 아이콘 생김 = 백그라운드 서버 동작 중)

### 방법 B — Homebrew
```bash
brew install --cask ollama
open -a Ollama       # 서버 실행
```

### 동작 확인
```bash
ollama --version
curl -s http://localhost:11434/api/tags
# 두 줄짜리 JSON ({"models":[]}) 같은 게 나오면 OK
```

> ❓ 포트 11434? Ollama 서버가 듣는 곳. 다른 프로그램이 쓰면 충돌.

## 3. Python + 라이브러리 (한 번만, 1분)

macOS 13+ 에는 `python3` 가 기본 포함. 버전만 확인:

```bash
python3 --version       # 3.9 이상이면 OK
```

### 필수 두 개 설치
```bash
python3 -m pip install --user mlx-lm
python3 -m pip install --user huggingface_hub
```

- `mlx-lm` = MLX로 모델 돌리고/변환하는 라이브러리
- `huggingface_hub` = HuggingFace에서 모델 다운로드하는 도구 (`huggingface-cli`)

### PATH 등록 (이게 없으면 `huggingface-cli not found`)
설치된 명령어가 `~/Library/Python/3.9/bin/` 에 들어가는데, 시스템이 그 경로를 모름. 알려줘야 함.

```bash
echo 'export PATH="$HOME/Library/Python/3.9/bin:$PATH"' >> ~/.zshrc
source ~/.zshrc

# 확인
which huggingface-cli       # /Users/.../bin/huggingface-cli 가 나오면 OK
```

> ❓ `~/.zshrc` 가 뭐예요? 터미널(zsh)이 시작될 때마다 읽는 설정 파일. PATH 같은 환경변수를 적어 두는 곳.

## 4. 모델 저장 폴더 만들기 (10초)

모든 MLX 모델은 `~/models/` 안에 둘 거예요:

```bash
mkdir -p ~/models
ls -la ~/models       # 빈 폴더 확인
```

> ❓ `mkdir -p` 의 `-p` 는 "중간 폴더가 없으면 같이 만들어"라는 뜻. 안전하게 두 번 실행해도 OK.

## 5. EXAONE 4.0 1.2B — Ollama 버전 (1~3분, 0.8GB)

```bash
ollama pull hf.co/LGAI-EXAONE/EXAONE-4.0-1.2B-GGUF:Q4_K_M
```

진행률이 쭉 올라가다 끝나면:

```bash
# 확인
ollama list | grep EXAONE

# 채팅 테스트
ollama run hf.co/LGAI-EXAONE/EXAONE-4.0-1.2B-GGUF:Q4_K_M
>>> 안녕! 너 누구야?
[모델 답변]
>>> /bye          # 종료
```

> ❓ `Q4_K_M` 이 뭐예요? 양자화 종류. `Q4` = 4비트, `K_M` = 그중에서도 균형 잡힌 변종. 품질 거의 그대로 + 크기 ¼.

## 6. EXAONE 4.0 1.2B — MLX 4bit (2~5분, 0.7GB)

같은 모델을 **MLX 포맷으로 변환**해서 더 빠르게 돌립니다.

```bash
python3 -m mlx_lm convert \
  --hf-path LGAI-EXAONE/EXAONE-4.0-1.2B \
  --mlx-path ~/models/exaone-4.0-1.2b-4bit-mlx \
  -q
```

- `--hf-path`: HuggingFace 레포 경로
- `--mlx-path`: 변환된 결과를 저장할 내 폴더
- `-q`: 4bit 양자화

끝나면 확인:

```bash
ls ~/models/exaone-4.0-1.2b-4bit-mlx
# config.json  model.safetensors  tokenizer.json  ...
```

채팅 테스트:

```bash
python3 -m mlx_lm chat --model ~/models/exaone-4.0-1.2b-4bit-mlx
>>> 한국어로 자기소개 해줘
[모델 답변, 매우 빠름 — 282 tok/s]
```
종료: `Ctrl + D`

> ⚠ 변환 도중 메모리 부족 에러? 다른 무거운 앱 닫고 재시도. 또는 큰 머신에서 변환 후 결과 폴더만 복사 가능.

## 7. SKT A.X 4.0 Light — MLX 4bit (15~30분, 14GB 다운 → 4.5GB)

> ⚠ **원본이 14GB라 다운로드가 깁니다.** 백그라운드로 두고 다른 일 하세요.

```bash
python3 -m mlx_lm convert \
  --hf-path skt/A.X-4.0-Light \
  --mlx-path ~/models/ax-4.0-light-4bit-mlx \
  -q
```

내부적으로:
1. HuggingFace 에서 BF16 원본을 `~/.cache/huggingface/` 로 다운로드 (14GB)
2. 4bit로 변환해서 `~/models/ax-4.0-light-4bit-mlx/` 에 저장 (4.5GB)

확인:

```bash
ls -la ~/models/ax-4.0-light-4bit-mlx
du -sh ~/models/ax-4.0-light-4bit-mlx     # 4.5G
```

채팅:

```bash
python3 -m mlx_lm chat --model ~/models/ax-4.0-light-4bit-mlx
```

### 디스크 회수 (선택)
변환이 끝나면 `~/.cache/huggingface/` 의 14GB 원본은 **지워도** 변환된 모델은 그대로 동작:

```bash
du -sh ~/.cache/huggingface     # 14GB+
rm -rf ~/.cache/huggingface     # 회수 (모델 동작에 영향 X)
```

## 8. Upstage Solar Pro 22B — Ollama (20~40분, 13GB)

Solar는 GGUF 파일로 받아 Ollama에 등록합니다.

### 8-1. GGUF 다운로드
```bash
mkdir -p ~/models/solar-pro
huggingface-cli download MaziyarPanahi/solar-pro-preview-instruct-GGUF \
  --include "solar-pro-preview-instruct.Q4_K_M.gguf" \
  --local-dir ~/models/solar-pro
```

확인:
```bash
ls -lh ~/models/solar-pro/
# solar-pro-preview-instruct.Q4_K_M.gguf  (13GB)
```

### 8-2. Modelfile 작성
"이 GGUF 파일을 `solar-pro` 라는 이름으로 등록" 레시피:

```bash
cat > /tmp/solar.modelfile <<EOF
FROM $HOME/models/solar-pro/solar-pro-preview-instruct.Q4_K_M.gguf
EOF

cat /tmp/solar.modelfile      # 내용 확인
```

### 8-3. Ollama에 등록
```bash
ollama create solar-pro -f /tmp/solar.modelfile
ollama list | grep solar
```

### 8-4. 채팅
```bash
ollama run solar-pro
>>> 파이썬으로 피보나치 수열 만들어줘
[모델 답변]
>>> /bye
```

> ⚠ 24GB 맥에서 Solar 띄우면 메모리 빡빡. Chrome/Slack/VS Code 닫는 게 좋음.

## 9. 단축 명령(PATH) 등록 — 옵션이지만 강추

매번 `python3 -m mlx_lm chat --model ~/models/ax...` 치기 귀찮죠. 이 레포의 `bin/` 폴더에 단축 스크립트가 있어요. PATH에 추가하면 어디서나 `ax`, `exaone-mlx` 같은 짧은 명령으로 실행됩니다.

```bash
# 이 레포가 어디 있는지 확인
cd ~/DEV/llmDev/korea-ai          # 본인 위치로 조정
pwd

# zshrc에 PATH 한 줄 추가
echo "export PATH=\"$PWD/bin:\$PATH\"  # korea-ai launchers" >> ~/.zshrc
source ~/.zshrc

# 확인
which ax           # /.../korea-ai/bin/ax
ax "안녕"          # 한 번 질문하고 끝
```

수동으로 ax 등을 만들고 싶으면 [bin/ax](../bin/ax) 등을 참고해서 본인 경로에 맞게 직접 작성해도 됩니다. 본질은:

```bash
#!/usr/bin/env bash
python3 -m mlx_lm chat --model ~/models/ax-4.0-light-4bit-mlx "$@"
```

이런 한 줄짜리 래퍼 스크립트입니다.

## 10. (옵션) 자동 정리 cron

매주 일요일 새벽 4시에 메모리 정리:

```bash
crontab -e
```
편집기가 열리면 맨 아래에 추가:
```
0 4 * * 0 /Users/mo/DEV/llmDev/korea-ai/maintain.sh --auto >> ~/.korea-ai-maint.log 2>&1
```
저장 후 종료. 확인:
```bash
crontab -l
```

> ❓ `0 4 * * 0` 형식? `분 시 일 월 요일`. 0=일요일.

## 🆘 자주 막히는 곳

### `huggingface-cli: command not found`
3단계 PATH 설정을 안 했음. `source ~/.zshrc` 하거나 새 터미널 열기.

### `python3 -m mlx_lm: No module named mlx_lm`
3단계 pip install 실패 또는 안 함. 다시:
```bash
python3 -m pip install --user mlx-lm
```

### `ollama: command not found`
2단계 Ollama 앱 설치 안 됨. 또는 brew 경로 PATH에 없음.
```bash
ls /Applications/Ollama.app
which ollama
```

### `Address already in use` (포트 11434)
이미 Ollama 서버가 떠 있음. 정상이거나, 충돌 시:
```bash
pkill -f ollama
open -a Ollama
```

### MLX 변환 중 OOM (메모리 부족)
다른 앱 닫고 재시도. 14GB 원본 변환은 메모리 많이 씀.

### A.X 다운로드가 멈춤
HF 캐시는 이어받기 지원. 같은 명령 다시 실행하면 멈춘 지점부터 계속.

### 디스크 부족
- `~/.cache/huggingface` 삭제 (변환 끝났으면 안전)
- `ollama rm <모델>` 으로 안 쓰는 Ollama 모델 제거

### 🪟 "Docker requires privileged access" 비밀번호 창이 뜸
맥 로그인 시 (특히 **Rosetta 비활성화 직후**) Docker Desktop 이 자동 실행되며 다음과 같은 창을 띄웁니다:

> 🔒 Docker — *Docker Desktop requires privileged access to configure privileged port mapping. Enter your password to allow this.*

**이 레포(korea-ai)는 Docker 를 전혀 안 씁니다.** 즉 이 창은 Docker Desktop 자체가 띄우는 별개 알림이에요. 대처:

- **Docker 가 꼭 필요한 경우** (다른 작업용) → 맥 로그인 비밀번호 입력 → OK. 정상 동작.
- **Docker 안 쓸 거면** → Cancel 누르고, 자동 실행 끄기:
  ```bash
  # 1) 메뉴바 고래 아이콘 → Settings → General →
  #    "Start Docker Desktop when you sign in" 체크 해제
  # 2) 또는 아예 종료/제거
  osascript -e 'quit app "Docker"'
  brew uninstall --cask docker        # 완전 제거
  ```
- 매번 비밀번호를 묻는 게 싫다면 → Settings → Advanced →
  *"Allow privileged port mapping"* 옵션을 끄거나 *"Configure..."* 에서 권한을 한 번만 부여.

> ❓ 왜 Rosetta 끄면 나타나나요? Rosetta 가 꺼지면 Docker 가 가상화 백엔드(VM 네트워킹)를 다시 셋업하면서 1024 미만 "특권 포트" 매핑 권한을 다시 요구합니다. 정상 동작입니다.

#### 그다음 "Finish setting up Docker Desktop" 창이 뜸 — 뭐 고르지?
비밀번호 입력 후 또는 업데이트 후 나오는 화면:
> *Complete the installation of Docker Desktop. The configurations below can be changed later in Settings.*
> ⚫ **Use recommended settings (requires password)**
> ⚬ Use advanced settings

- **이 레포(korea-ai) 사용자**: 아무거나 고르고 Finish 입니다. 이 레포는 Docker 를 안 씁니다. (더 이상 알림이 뜨기 싫으면 Docker 자동시작을 끄세요 — 위 안내 참고)
- **Docker 도 같이 쓸 계획**: **Use recommended settings → Finish** (기본값). 대부분의 개발 용도에 맞은 설정을 자동으로 잡아주고, 나중에 Settings 에서 언제든 바꿀 수 있습니다.
- **Use advanced settings** 는 아래 세 가지를 직접 선택하고 싶을 때만:
  1. 특권 포트(<1024) 매핑 허용 여부
  2. `/usr/local/bin` 에 docker CLI 심볼링크 설치 여부
  3. 도커 소켓 `/var/run/docker.sock` 생성 여부
  잘 모르면 그냥 recommended 로 가세요.

> 📝 recommended 를 고르면 발생하는 "requires password" = 앞서 본 특권 포트 권한 설정을 위해 한 번 더 맥 로그인 비밀번호를 물어봅니다. 입력하면 끝이에요.

## 🔁 완전 제거 (수동)

```bash
# 1. Ollama 모델
ollama rm hf.co/LGAI-EXAONE/EXAONE-4.0-1.2B-GGUF:Q4_K_M
ollama rm solar-pro

# 2. MLX 모델
rm -rf ~/models/exaone-4.0-1.2b-4bit-mlx
rm -rf ~/models/ax-4.0-light-4bit-mlx
rm -rf ~/models/solar-pro

# 3. HF 캐시
rm -rf ~/.cache/huggingface

# 4. PATH 등록 제거
sed -i '' '/korea-ai/d' ~/.zshrc
sed -i '' '/Library\/Python\/3.9\/bin/d' ~/.zshrc

# 5. (선택) Ollama 자체 제거
brew uninstall --cask ollama   # 또는 Ollama.app 을 휴지통으로
rm -rf ~/.ollama

# 6. (선택) Python 패키지
python3 -m pip uninstall mlx-lm huggingface_hub

# 7. cron 해제
crontab -l | grep -v korea-ai | crontab -
```
