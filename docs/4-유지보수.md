# 4. 유지보수 (자동화)

## 한눈에

| 작업 | 명령 | 언제 |
|---|---|---|
| 메모리 강제 정리 | `./launcher.sh /unload` 또는 `ai-stop` | 다른 무거운 앱 쓰기 전 |
| 디스크 캐시 정리 | `ai-clean` | 한 달에 한 번 |
| 깊은 정리 (HF 캐시까지) | `./maintain.sh --deep` | 디스크 부족할 때 |
| 상태 점검 | `ai-status` | 의심 갈 때 |
| **자동 (cron)** | `./install.sh` 가 등록 | 한 번만 셋업 |

---

## 수동 유지보수

### 1) 메모리 비우기
```bash
./launcher.sh /unload          # 메뉴에서: > /unload
ai-stop                         # 어디서나
ai-stop --server                # Ollama 서버까지 종료 (메모리 완전 해제)
```

### 2) 디스크 정리 (안전)
```bash
ai-clean                        # 임시/락 파일만 삭제 (모델 보존)
ai-clean --dry-run              # 보기만
ai-clean --hf-cache             # HF 캐시(~13GB BF16 원본) 까지 삭제
                                # 변환된 모델은 ~/models/ 에 안전하게 남음
```

### 3) 통합 유지보수
```bash
./maintain.sh                  # 대화형 (확인하며 진행)
./maintain.sh --status         # 점검만, 변경 없음
./maintain.sh --deep           # HF 캐시 삭제 포함
```

`maintain.sh` 가 하는 일:
1. Ollama 로드된 모델 → 언로드
2. 떠있는 mlx 프로세스 → 종료
3. Solar GGUF 다운로드 끝났는데 Ollama 미등록이면 → 자동 등록
4. 모델 폴더의 `*.incomplete` `*.lock` `*.tmp` → 삭제
5. (--deep) HF 캐시 삭제
6. 디스크/메모리 리포트

## 자동화 (cron)

### 설치 시 등록
`./install.sh` 마지막 단계에서 등록 여부를 묻습니다.

### 수동 등록
```bash
crontab -e
# 아래 줄 추가 (매주 일요일 04:00)
0 4 * * 0 /Users/mo/DEV/llmDev/korea-ai/maintain.sh --auto >> ~/.korea-ai-maint.log 2>&1
```

### 등록 확인
```bash
crontab -l | grep maintain
tail -20 ~/.korea-ai-maint.log
```

### 등록 해제
```bash
crontab -l | grep -v maintain.sh | crontab -
```

> 💡 cron의 `--auto` 모드는 **메모리 정리만** 하고 HF 캐시는 보존합니다 (예측 가능 + 안전).

## 모델 업데이트

### Ollama 모델
```bash
ollama pull hf.co/LGAI-EXAONE/EXAONE-4.0-1.2B-GGUF:Q4_K_M
ollama pull solar-pro                                          # (재등록은 install.sh)
```

### MLX 모델 (재변환)
```bash
rm -rf ~/models/ax-4.0-light-4bit-mlx
./install.sh                    # 누락 감지 → 다시 변환
```

### 새 모델 라인업이 나오면
- EXAONE 7.8B / 32B 가 나오면 `~/models/` 경로 추가하고 `bin/exaone7b` 같은 단축 명령 추가
- A.X 4.0 정식판이 나오면 `--hf-path skt/A.X-4.0` 으로 재변환

## 백업 / 복구

### 모델 백업
```bash
# 외장 디스크에 복사 (재다운로드 안 해도 됨)
rsync -av --progress ~/models/ /Volumes/Backup/models/
```

### 다른 맥으로 옮기기
```bash
# 옛 맥
rsync -av ~/models/ user@new-mac:~/models/
rsync -av ~/DEV/llmDev/korea-ai/ user@new-mac:~/DEV/llmDev/korea-ai/

# 새 맥
cd ~/DEV/llmDev/korea-ai
./install.sh --check            # 모델은 이미 있으니 PATH/cron만 등록됨
source ~/.zshrc
ai-status
```

### Ollama 모델은
`~/.ollama/models/` 에 들어 있어요. 같이 백업하면 재 pull 불필요.

## 완전 제거

```bash
# 1. 메모리 정리
ai-stop --server

# 2. cron 해제
crontab -l | grep -v maintain.sh | crontab -

# 3. PATH 등록 제거
sed -i '' '/korea-ai/d' ~/.zshrc

# 4. 모델 삭제
rm -rf ~/models/exaone-4.0-1.2b-4bit-mlx
rm -rf ~/models/ax-4.0-light-4bit-mlx
rm -rf ~/models/solar-pro
ollama rm hf.co/LGAI-EXAONE/EXAONE-4.0-1.2B-GGUF:Q4_K_M
ollama rm solar-pro

# 5. (선택) 폴더 자체 삭제
rm -rf ~/DEV/llmDev/korea-ai
```
