# 00. 완전 기초 — 로컬 LLM이 뭔가요?

**🌐 Language**: **한국어 (현재)** · [English](en/00-basics.md)

> "AI가 뭔지는 들어봤는데, 로컬 LLM? MLX? Ollama? 양자화? 토큰? …" 하나도 몰라도 됩니다.
> 이 문서는 **컴퓨터 잘 모르는 분도** 따라올 수 있게 모든 용어를 처음부터 설명합니다.
> 끝까지 읽으면 이 레포가 왜 필요하고, 무엇을 하는지 자연스럽게 이해됩니다.

---

## 0. 우선, 머릿속 그림 한 장

```
   ┌────────────────────────────────────────────────────────┐
   │                                                        │
   │   당신의 질문 → [ AI 모델 ] → 답변                     │
   │                                                        │
   └────────────────────────────────────────────────────────┘

   ChatGPT / Claude / Gemini :
       질문 → [인터넷] → 미국 서버의 거대한 AI → [인터넷] → 답변
       (외부 회사가 내 질문을 봅니다)

   로컬 LLM (이 레포):
       질문 → [내 맥북 안의 AI] → 답변
       (인터넷도 안 쓰고, 외부에 안 나갑니다)
```

이 레포는 두 번째 그림을 만드는 도구입니다. **끝.**

이제 각 단어가 정확히 뭔지 풀어봅니다.

---

## 1. AI / 모델 / LLM 이 뭐예요?

### AI (인공지능, Artificial Intelligence)
사람처럼 "판단·이해·생성"을 하는 컴퓨터 프로그램의 총칭. 너무 넓은 단어라서 보통 **종류로 나눠서** 부릅니다.

### 모델 (Model)
AI의 "뇌"에 해당하는 **거대한 숫자 덩어리 파일**. 우리가 흔히 "AI"라고 부르는 건 사실 이 모델 파일입니다.
- 크기는 수백 MB ~ 수백 GB
- 회사들이 슈퍼컴퓨터로 몇 달간 학습시켜서 만듦
- **한 번 만들면 우리는 그냥 다운받아 사용**

### LLM (Large Language Model, 거대 언어 모델)
**글을 이해하고 글로 답하는** 모델. ChatGPT가 대표적인 LLM입니다.
- 입력: 글(문장, 질문, 코드)
- 출력: 글(답변, 요약, 번역)
- 그림 그리거나 음성 인식하지 않음 (그건 다른 종류의 모델)

> 💡 **이 레포가 다루는 건 모두 LLM**입니다 — EXAONE, A.X, Solar 모두 글로 입력받아 글로 답하는 모델.

---

## 2. 그럼 "로컬 LLM"은 뭐예요?

### 로컬 (Local) = "내 컴퓨터에서"
"로컬 LLM" = **내 컴퓨터 안에서 돌아가는 LLM**.

### 클라우드 LLM vs 로컬 LLM 비교

| | 클라우드 LLM (ChatGPT 등) | 로컬 LLM (이 레포) |
|---|---|---|
| **돌아가는 곳** | 미국·해외의 거대한 데이터센터 | 내 맥북 |
| **인터넷** | 필수 | 불필요 (한 번 받으면) |
| **데이터** | 외부 서버로 전송됨 | 내 컴퓨터 밖으로 안 나감 |
| **비용** | 보통 월 구독 ($20+) | 모델 무료 (저장공간만 씀) |
| **속도** | 빠름 (강력한 서버) | 컴퓨터 사양에 따라 다름 |
| **품질** | 매우 높음 (수천억 매개변수) | 그보다 낮음 (수십억) |
| **차단/검열** | 회사 정책에 좌우 | 내 마음대로 |
| **인터넷 끊겼을 때** | 못 씀 | 정상 동작 |

> 💡 **둘 다 장단점이 있어서**, 보통 같이 씁니다. 민감한 자료·오프라인·일상 용도는 로컬, 정말 똑똑한 추론이 필요할 땐 ChatGPT.

---

## 3. 왜 로컬 LLM을 쓸까요? — 5가지 이유

### ① 프라이버시
- 회사 기밀, 개인정보, 의료기록을 ChatGPT에 입력하면 **외부 서버에 그대로 전송**됩니다
- 로컬 LLM은 인터넷도 안 쓰니 데이터가 새지 않음

### ② 오프라인 동작
- 비행기 안, 지하실, 보안망 환경, 인터넷 안 되는 곳에서도 동작
- 모델은 한 번만 다운로드하면 끝

### ③ 비용
- ChatGPT Plus = 월 $20 (~3만 원)
- 로컬 LLM = **모델 무료** (전기료만 미세하게)

### ④ 속도
- 짧은 질문은 로컬이 더 빠를 수 있음 (네트워크 왕복 시간 없음)
- 단, 큰 모델은 사양에 따라 느릴 수 있음

### ⑤ 우리말 / 도메인 특화
- 한국 회사가 만든 모델은 **한국어를 더 잘** 이해 (한국어 토크나이저, 한국 데이터로 학습)
- 의료/법률/금융 등 특정 도메인에 튜닝된 모델도 많음

## 4. 자주 나오는 용어 미리 정리

### 토큰 (Token)
모델이 글을 처리하는 **최소 단위**. 영어 단어 하나가 보통 1토큰, 한국어는 음절·조각 단위로 쪼개져 1~3토큰.
- "안녕하세요" → 약 3~4 토큰
- "Hello" → 1 토큰
- 모델은 "한 번에 토큰 X개 처리 가능" 같은 한계가 있음 (= 컨텍스트 길이)

### 매개변수 (Parameter, 파라미터)
모델 안에 있는 "뉴런 연결 강도 숫자"의 개수. **클수록 똑똑하지만 무거움**.
- EXAONE 1.2B = 12억 개
- Solar 22B = 220억 개
- GPT-4 (추정) = 1조 개 이상
- B = Billion (10억)

### 양자화 (Quantization)
모델 파일을 **압축**해서 메모리·디스크를 적게 쓰는 기법. 약간의 품질 손실로 크기를 1/2 ~ 1/4 로 줄입니다.
- BF16 = 원본 (한 숫자 = 16비트)
- Q4 / 4bit = 4비트로 압축 (4배 작아짐, 품질 거의 비슷)
- 이 레포는 거의 **4bit 양자화 모델**을 사용 (메모리 절약)

### 컨텍스트 길이 (Context Length)
모델이 한 번에 기억할 수 있는 글의 길이. 보통 토큰 단위.
- A.X 4.0: 32,768 토큰 (= 책 한 권 정도의 한국어)
- 짧으면 긴 문서 요약 못 함

### 추론 (Inference)
모델이 **답을 만들어내는 과정**. "학습(training)"과 반대 개념.
- 학습: 회사가 슈퍼컴으로 모델을 만드는 단계 (수십억 원)
- 추론: 우리가 만든 모델로 답을 받는 단계 (개인 컴퓨터로 가능)

> 💡 **이 레포는 추론만** 합니다. 모델은 LG·SKT·Upstage가 이미 학습시켜 놓은 걸 가져다 씁니다.

## 5. 도구 이름 풀이 — Ollama / MLX / Hugging Face

로컬 LLM을 돌리려면 **모델 파일** + **그걸 실행할 프로그램**이 둘 다 필요합니다. 자주 등장하는 도구들:

### Ollama
- **로컬 LLM을 가장 쉽게 돌리게 해주는 프로그램**
- macOS / Windows / Linux 모두 지원
- 명령어 한 줄로 모델 다운로드·실행
- 비유: "AI 모델용 앱스토어 + 실행기"
- 공식: [ollama.com](https://ollama.com)

### MLX (Apple)
- **Apple Silicon(M1~M5)에서 AI를 빠르게 돌리는 프레임워크**
- Apple이 직접 만들어 통합 메모리 구조에 최적화
- 같은 모델이라도 MLX로 돌리면 훨씬 빠름 (M5 Pro에서 282 토큰/초)
- macOS 전용 (Apple Silicon만)
- 공식: [github.com/ml-explore/mlx](https://github.com/ml-explore/mlx)

### Hugging Face
- **모델 파일들의 GitHub**, AI 모델을 모아둔 사이트
- LG도, SKT도, Upstage도 여기에 모델을 공개해 둠
- `huggingface-cli` 명령으로 다운로드
- 공식: [huggingface.co](https://huggingface.co)

### GGUF / safetensors
- 모델 **파일 포맷** 이름
- GGUF = Ollama가 쓰는 포맷
- safetensors = MLX·HuggingFace가 쓰는 포맷
- 같은 모델을 두 포맷으로 받을 수 있음 (이 레포는 모델 종류에 따라 적절한 걸 선택)

## 6. 통합 메모리 / 4bit / 282 토큰/초 — 왜 빠른지

### Apple Silicon의 통합 메모리 (Unified Memory)
일반 PC: CPU 메모리와 GPU 메모리가 분리 → 데이터를 계속 복사해야 함 (느림)
M1~M5: CPU·GPU·메모리가 한 칩 → **복사 없이 바로 접근** (빠름)

### 4bit 양자화 + MLX = 폭발적 속도
- 모델을 4bit로 압축 → 메모리 사용량 1/4
- MLX가 통합 메모리로 즉시 접근
- 결과: **노트북에서 ChatGPT 같은 속도**

### 실측 (M5 Pro 24GB)
- EXAONE 1.2B (MLX 4bit): **282 토큰/초** (=초당 한국어 약 200자)
- A.X 4.0 Light (MLX 4bit): 약 65 토큰/초
- Solar 22B (Ollama Q4): 약 10~15 토큰/초

> 💡 사람이 글 읽는 속도가 보통 5~10 토큰/초니 **눈으로 따라가기 벅찰 정도로 빠릅니다.**

## 8. 자주 묻는 기초 질문

### Q. 로컬 LLM이 진짜 ChatGPT 만큼 똑똑해요?
A. **아닙니다.** 매개변수 차이가 큽니다(GPT-4 ~1조 vs 로컬 1.2B~22B). 다만 일상 한국어 질문·요약·메일 작성은 충분합니다.

### Q. 인터넷 한 번도 안 써요?
A. **모델을 처음 다운로드할 때만** 필요. 그 이후는 완전히 오프라인.

### Q. 24GB 맥북에 22B 모델이 어떻게 들어가요?
A. **양자화** 덕분. 원본 BF16은 44GB지만 4bit로 압축하면 13GB로 줄어듭니다.

### Q. Ollama랑 MLX 둘 다 깔아야 해요?
A. **네**, 모델마다 최적의 도구가 달라서. 이 레포의 `install.sh`가 자동으로 둘 다 설정해 줍니다.

### Q. 모델 파일은 어디에 저장돼요?
A. `~/models/` 폴더(MLX) 와 `~/.ollama/models/` 폴더(Ollama). 합쳐서 약 25GB.

### Q. 모델이 답을 잘못하면?
A. 모든 LLM은 **환각(hallucination)** 을 일으킵니다. 자신 있게 틀린 답을 줄 수 있으니 중요한 사실은 항상 검증하세요.

### Q. "1.2B는 작아서 멍청하다"던데, 그럼 왜 써요?
A. **속도와 효율** 때문. 빠른 단답·요약·간단한 번역은 1.2B로도 충분하고, 메모리도 1GB 미만이라 부담 없음. 어려운 일은 22B 모델을 띄우면 됨.

### Q. 한국 소버린 AI가 왜 중요해요?
A. 한국어 데이터·기술·운영을 외부에 의존하지 않기 위함. → 자세히는 [docs/0-소개.md](0-소개.md)
