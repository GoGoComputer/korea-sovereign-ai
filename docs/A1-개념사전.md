# 1. 컴퓨터 기본

## 터미널 (Terminal)

**검은 화면에 글자만 입력해서 컴퓨터를 조작하는 앱.**

- macOS 기본 앱: `응용 프로그램 → 유틸리티 → 터미널.app`
- 빠르게 열기: `Cmd + Space` → "터미널" 입력 → Enter
- 입력하는 글자 = "명령어(command)"
- 마우스 안 쓰고 키보드만 씀

**왜 써요?** AI 모델 실행, git, 설치 같은 작업이 마우스보다 빠르고 자동화하기 쉬워서.

```
첫 화면 예시:
mo@MacBook ~ %
└─ 사용자명  컴퓨터명  현재폴더  입력대기
```

마지막의 `%` (혹은 `$`) 는 "여기에 명령어 입력하세요" 라는 표시. 입력 후 Enter.

## 폴더 (Folder, Directory)

**파일들을 담는 상자.** 윈도우에서 보던 노란 폴더 그것.

- 영어로 directory(디렉토리)라고도 함 (같은 말)
- 폴더 안에 폴더가 들어갈 수 있음 → 트리 구조

### 터미널에서 폴더 다루기

| 하고 싶은 일 | 명령어 | 예시 |
|---|---|---|
| 지금 어느 폴더? | `pwd` | `/Users/mo` |
| 이 폴더 안에 뭐 있나? | `ls` | (파일 목록) |
| 다른 폴더로 이동 | `cd 폴더이름` | `cd Documents` |
| 한 단계 위로 | `cd ..` | |
| 홈 폴더로 | `cd ~` 또는 그냥 `cd` | |
| **새 폴더 만들기** | `mkdir 폴더이름` | `mkdir models` |
| **여러 단계 한 번에 만들기** | `mkdir -p a/b/c` | (a 안에 b 안에 c) |
| 폴더 삭제 (빈 것) | `rmdir 폴더이름` | |
| 폴더 + 내용 통째 삭제 ⚠ | `rm -rf 폴더이름` | (위험! 신중히) |

### Finder (Mac 파일 탐색기)에서 만들기
1. Finder 열기 → 만들 위치로 이동
2. `Cmd + Shift + N` (새 폴더 단축키)
3. 이름 입력 → Enter

## 경로 (Path)

**파일·폴더가 어디 있는지 표시하는 주소.**

```
/Users/mo/DEV/llmDev/korea-ai/README.md
└── 루트  사용자  내폴더 ...           파일
```

- `/` 로 시작 = "절대 경로" (어디서 입력해도 같은 곳)
- `./` 또는 그냥 이름 = "상대 경로" (지금 위치 기준)

### `~` (틸드)
**"내 홈 폴더"의 줄임말.**
- `~` = `/Users/mo` (당신의 사용자 홈)
- `~/models` = `/Users/mo/models`
- `~/.zshrc` = `/Users/mo/.zshrc`

### 숨김 폴더/파일
이름이 `.` 으로 시작하면 숨김(Finder에서 안 보임).
- 예: `~/.ollama/`, `~/.zshrc`
- Finder에서 보이게: `Cmd + Shift + .` (점)
- 터미널에서 보기: `ls -a`

---

## 파일 확장자

이름 끝의 점 + 영문 = 파일 종류 표시.

| 확장자 | 뜻 | 이 레포에서 |
|---|---|---|
| `.md` | Markdown 문서 | README.md, docs/* |
| `.sh` | shell script (실행 가능) | install.sh, launcher.sh |
| `.py` | Python 코드 | bin/mlx-chat |
| `.gguf` | Ollama용 모델 파일 | solar-pro-...gguf |
| `.safetensors` | MLX/HF용 모델 파일 | model.safetensors |
| `.json` | 설정 파일 | config.json |

## 명령어 / 인자 / 옵션

```
ollama pull EXAONE-4.0-1.2B
└─명령어 └서브명령 └인자(arg)
```

```
./install.sh --yes --minimal
└─실행파일    └옵션  └옵션
```

- `--xxx` 또는 `-x` 로 시작 = 옵션(option, flag)
- 그냥 단어 = 인자(argument)

### 자주 쓰는 키
- `Tab` = 자동완성 (반쯤 친 후 누르면 나머지 채워줌)
- `↑` / `↓` = 이전 명령어 다시 꺼내기
- `Ctrl + C` = 실행 중단
- `Ctrl + D` = 입력 종료 (MLX 채팅 끝낼 때 사용)
- `Cmd + K` = 화면 지우기

# 2. 다운로드 / 설치 도구

## Homebrew (`brew`)
**macOS의 "프로그램 자판기".** 한 줄로 프로그램 설치.

```bash
brew install git              # git 설치
brew install --cask ollama    # Ollama 앱 설치
```
공식: <https://brew.sh>

> 💡 이 레포의 `web-install.sh` 는 brew 없이도 동작하지만, brew가 있으면 편합니다.

---

## pip (Python 패키지 관리자)
**Python 라이브러리(코드 모음)를 설치.**

```bash
python3 -m pip install --user mlx-lm
python3 -m pip install --user huggingface_hub
```
- `--user` = 내 계정에만 설치 (관리자 권한 불필요)
- 이 레포는 `mlx-lm` (MLX 추론), `huggingface_hub` (HF 다운로드) 두 개를 씀

---

## git
**소스코드 버전 관리 도구.** GitHub에서 코드 받을 때 필수.

```bash
git --version                  # 설치 확인
git clone <URL>                # 코드 받기
git pull                       # 최신화
git status                     # 현재 상태
```
없으면 `xcode-select --install` 로 설치.

---

## curl
**인터넷에서 파일/데이터 받는 명령어.** 거의 모든 OS에 기본 내장.

```bash
curl -fsSL <URL> | bash       # 받아서 바로 실행 (이 레포의 한 줄 설치)
```
- `-f` = 실패 시 조용히 종료
- `-s` = 진행률 숨김
- `-S` = 에러는 표시
- `-L` = 리다이렉트 따라감

---

## xcode-select
Apple의 **개발자 기본 도구 모음(Command Line Tools)** 설치 명령어.

```bash
xcode-select --install         # 창이 뜨면 [설치] 클릭, 5~10분 대기
```
이걸 깔면 `git`, `make`, `clang` 같은 게 들어옴. **이 레포 사용 전 사실상 필수.**

## Hugging Face 🤗

**AI 모델·데이터셋의 "GitHub".** 전 세계 사람들이 모델을 공개·다운로드하는 사이트.

- 웹: <https://huggingface.co>
- LG, SKT, Upstage 모두 여기에 한국 모델 공개해 둠
- 회원가입 없이도 받을 수 있음 (일부는 약관 동의 필요)

### 핵심 용어
- **레포(repository)**: 모델 하나가 들어 있는 폴더 (예: `LGAI-EXAONE/EXAONE-4.0-1.2B`)
- **모델 카드**: 그 모델의 README (스펙·라이선스·사용법)
- **`huggingface-cli`**: 터미널에서 다운로드하는 도구

```bash
# A.X 4.0 Light 모델 전체 다운로드
huggingface-cli download skt/A.X-4.0-Light --local-dir ~/models/ax-original
```

> ⚠️ 모델은 보통 **수 GB ~ 수십 GB**. 디스크 공간·인터넷 시간 주의.

---

## Ollama 🦙

**로컬 AI를 가장 쉽게 돌리는 프로그램.**

- 공식: <https://ollama.com>
- macOS 앱으로 다운로드 + 설치 → 실행
- 백그라운드에서 "서버"가 떠 있고 (`localhost:11434`), 명령어로 모델을 부름

```bash
ollama --version              # 설치 확인
ollama pull llama3            # 모델 받기
ollama run llama3             # 채팅 시작
ollama list                   # 받은 모델 목록
ollama ps                     # 지금 메모리에 떠 있는 모델
ollama stop llama3            # 메모리에서 내림
ollama rm llama3              # 디스크에서 삭제
```

### Modelfile
Ollama만의 "모델 정의 레시피". GGUF 파일을 등록할 때 씀.

```
FROM /Users/mo/models/solar-pro/solar-pro-...gguf
```
이 한 줄을 파일로 저장하고 `ollama create solar-pro -f 그파일` → 등록 끝.

### 모델 저장 위치
`~/.ollama/models/` 안에 들어감. 백업하면 재 pull 불필요.

---

## MLX / mlx-lm

**Apple이 만든 Apple Silicon 전용 AI 프레임워크.**

- 공식: <https://github.com/ml-explore/mlx>
- M1~M5 의 통합 메모리 구조에 최적화 → 같은 모델도 훨씬 빠름
- Python 라이브러리 형태 (`pip install mlx-lm`)

```bash
# HuggingFace 모델을 MLX 4bit로 변환 (최초 1회)
python3 -m mlx_lm convert \
  --hf-path skt/A.X-4.0-Light \
  --mlx-path ~/models/ax-4.0-light-4bit-mlx -q

# 변환된 모델로 채팅
python3 -m mlx_lm chat --model ~/models/ax-4.0-light-4bit-mlx
```

> 💡 변환은 "원본(BF16, 큼) → 4bit(작음)" 압축 작업. 한 번 하면 그 결과를 계속 씀.

## 모델 파일 포맷 비교

| 포맷 | 사용처 | 특징 |
|---|---|---|
| **`.safetensors`** | HuggingFace, MLX | 안전한 텐서 저장, 모델 폴더에 여러 파일 (config.json + model.safetensors + tokenizer.json …) |
| **`.gguf`** | Ollama, llama.cpp | 단일 파일, 양자화 정보 내장 (Q4_K_M 등) |
| `.bin` (옛날) | PyTorch 옛 포맷 | 잘 안 씀, 보안 이슈로 safetensors로 옮겨감 |

같은 모델이 두 포맷 다 있는 경우가 많음. 도구에 맞춰 골라 받음.

# 4. 개념 한 번 더

(자세한 건 [00-기초.md](00-기초.md) 에 있음. 여기는 빠른 사전.)

| 용어 | 한 줄 |
|---|---|
| **모델 (Model)** | AI의 뇌 = 거대한 숫자 파일 |
| **LLM** | 글로 입력받아 글로 답하는 AI 모델 |
| **토큰 (Token)** | 모델이 글을 처리하는 최소 단위 (한국어 1글자 ≈ 1~3토큰) |
| **매개변수 (Parameter)** | 모델 안 "뉴런 연결 강도" 개수 (B = 10억) |
| **양자화 (Quantization)** | 모델을 4bit 등으로 압축 → 메모리 ¼ |
| **컨텍스트 길이** | 한 번에 기억 가능한 토큰 수 |
| **추론 (Inference)** | 모델로 답을 만드는 작업 (= 우리가 하는 일) |
| **학습 (Training)** | 모델을 새로 만드는 작업 (= LG/SKT가 하는 일) |
| **프롬프트** | 모델에게 주는 입력 (질문·지시) |
| **환각 (Hallucination)** | 모델이 자신 있게 틀린 답을 하는 현상 |

# 5. 파일·폴더가 실제로 어디 있나

```
~ (= /Users/내사용자명)
├── DEV/llmDev/korea-ai/        ← 이 레포 (스크립트들)
├── models/                     ← MLX 변환된 모델들 (수동 생성)
│   ├── exaone-4.0-1.2b-4bit-mlx/
│   └── ax-4.0-light-4bit-mlx/
├── .ollama/                    ← Ollama가 자동 생성
│   └── models/                  └── 받은 GGUF / blobs
├── .cache/huggingface/         ← HF가 자동 생성 (BF16 원본 캐시)
└── .zshrc                      ← 터미널 설정 (PATH 등록 위치)
```

> 💡 디스크가 빡빡할 땐:
> - `du -sh ~/models/*` — MLX 모델 크기 확인
> - `du -sh ~/.cache/huggingface` — HF 캐시 크기 (변환 후 지워도 OK)
> - `du -sh ~/.ollama/models` — Ollama 모델 크기
